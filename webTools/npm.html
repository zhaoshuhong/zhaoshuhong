# npm 发布流程

1.npm官网创建npm账户

npm网站地址：https://www.npmjs.com/

npm网站注册地址：https://www.npmjs.com/signup 会有邮箱的验证

2.登陆 npm login 登陆时候 username前 要加 ~ 否则会报错

3.如果 publish 失败

  • 3.1 404 从新npm adduser 注册 登陆
  • 3.2 404 切换npm 源 npm config set registry http://registry.npmjs.org
  • 3.3 403 邮箱验证
  • 3.4 403 You do not have permission to publish 需要更换 package.json的name

4发布成功 5.版本更新

npm version <update_type> -m "" 其中update_type 有三种：

  • patch增加一位补丁号（比如 1.1.1 -> 1.1.2）

  • minor增加一位小版本号（比如 1.1.1 -> 1.2.0）

  • major增加一位大版本号（比如 1.1.1 -> 2.0.0）

比如：

npm version patch -m "Version %s - v1.0.2"

注意

npm的package.json 不用改动 npm 命令会自动给版本号加一 改了会出错

最后提交更新的版本：

npm publish

6.查看npm的版本号 npm view <文件名> versions 结果: [ '0.0.1', '0.0.2' ]

7.npm 包的废弃和删除

  • 废弃: npm deprecate <package-name>[@version] <message> 例子: npm deprecate util-cli-zsh@1.0.0 'test deprecate'
  • 删除 npm unpublish <package-name> --force 删除的版本24小时后方可发布/只有发布72小时之内的包可以删除

# npm 模块中scope

scope是一种把相关的模块组织到一起的一种方式，也会在某些地方影响npm对模块的处理。npm公共仓库支持带有scope的的模块，同时npm客户端对没有scope的模块也是向后兼容的，所以可以同时使用两者。

scope在模块name中使用时，package.json中 "name": "@somescope/somepackagename" or npm init –scope=somescope 如果要发布一个公共socpe模块，你必须在最开始发布的时候指定–access public。 npm publish --access=public。

如果要发布私有模块，那么你必须有一个npm私有模块账户，可以选择自己搭建一个npm服务，或者直接使用官方的。发布私有模块的命令：

npm publish

# or
npm publish –access restricted

# npm 源管理

npm install -g nrm nrm:是npm的镜像源管理工具 nrm ls 查看可选源列表 nrm use taobao 切换源地址 nrm add <registry> <url> 增加定制源 nrm del <registry> 删除对应的源 nrm test <registry> 测试相应源响应时间

注意

使用时候 依然是 npm 指令来处理 npm install
