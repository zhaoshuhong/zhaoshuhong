# 2. GIT基础配置信息

  • 查看git 用户配置 cat ~/.gitconfig // 全局 cat .git/config // 当前项目
  • 配置全局用户 git config --global user.name "用户名" git config --global user.email "git账号"
  • 配置别名 git config --global alias.br branch
  • 删除全局配置 git config --global --unset alias.** git config --global --unset user.**

# 3. GIT常用指令

# git log

  • 查看本地git命令历史 git reflog
  • 查看本地git关联信息/远端所有关联 git branch -vv git remote show origin
  • 查看当前项目git SHA1 git rev-parse head

# git push

  • 本地分支和远程分支进行关联(git push origin branchName === git push) git push -u origin branchName
  • git push <远程主机名> <本地分支名>:<远程分支名>
  • git push -f origin 强制提交

# git branch

  • git branch -r | -a | -av 查看本地分支/远程分支/本地+远程/查看所有分支并带有最新的commit
  • git checkout -b dev master 基于 master 分支创建一个 dev 分支，并切换到该分支
  • git branch -m branchname 修改当前分支名
  • git brnach -d branchname 删除本地分支(-D 强制删除)

# git stash

将所有未提交的修改保存至堆栈中(pop:应用后删除记录/apply:应用后不删除记录/drop:直接删除记录)

  • git stash save "储存" git stash pop "stash@{index}"

# git tag

  • git tag v1.0 在HEAD上创建一个标签
  • git tag -a v0.1 -m "version 0.1 released" 添加标签说明
  • git push origin <tagname> / git push origin --tags 推送本地某个标签(or全部)至远端
  • git tag -d v0.1 删除一个本地标签
  • git push origin :refs/tags/<tagname> 删除远端标签(要先删除本地相应标签)

# git 删除文件

  • git rm filename 删除暂存区和工作区的文件
  • git rm --cached filename 只删除暂存区的文件, 不会删除工作区的文件

# git merge

  • git pull 分支名(要合并branch)
  • git merge --no-ff 禁止快进式合并:会生成一个新的提交，让当前分支的提交历史不会那么乱；
  • git merge --abort 接触合并状态

# 3. git 合并/提交/回退 问题 checkout/rebase/reset/revert/cherry-pick

checkout 会撤回到 add/commit 版本控制的文件历史
git checkout <commit_id> 切换到摸个指定的 commit版本 切换后会处于游离状态 不会对当前的版本产生影响 用于测试bug问题
reset --hard/soft <commit_id> 会回退版本更改HEAD指针 hard:会完全回退(工作区/暂存区/本地仓库) soft:回退 暂存区/本地仓库
rebase 使用rebase命令可以使我们的提交历史干净、简洁

注意

不要通过rebase对任何已经提交到公共仓库中的commit进行修改（你自己一个人玩的分支除外 -f）, push报错原因: push 会做简单的fastforword合并,对比commit变化, 但rebase会改变基点是本地和远端产生差异

通常操作 确保本地分支的commit_id是从要合并的分支拉出的.(如果不是:可通过 git reset --soft commit_id(old) 回退到) rebase branchName 有冲突解决 git rebase --continue 继续
git rebase -i 合并提交commit git revert <commit_id> 生成一个撤销指定提交版本的新提交 git cherry-pick <commit_id> 将制定的提交 commit应用于当前分支

git用户名密码输入错误

在控制面板里找到凭据管理器，选中 Windows 凭据，找到 git 的凭据，点击编辑，输入所用 github 的正确用户名和密码。

# 4.git 忽略已提交的文件

git update-index --assume-unchanged <path/fileName> 把不想提交的文件忽略
git update-index --no-assume-unchanged <path/fileName> 恢复跟踪
git ls-files -v 查询忽略的列表 开头为标识 小h的文件是 本地提交忽略的文件 更多 (opens new window)

# 5.git 规范

# git分支规范

  1. feature 新功能分支
  2. develop 开发分支
  3. test 测试分支
  4. release 预发布分支
  5. issue-** bug 分支
  6. master 发布分支(线上)

# git 提交规范

每次提交，Commit message 都包括三个部分：Header，Body 和 Footer，其中，Header 是必需的，Body 和 Footer 可以省略。

<type>(<scope>): <subject>   // type（必需）、scope:影响范围（可选）和subject:概述（必需）
<BLANK LINE>
<body>
<BLANK LINE>
<footer>    // BREAKING CHANGE:不兼容变动说明    Close #issue-number,issue-number   revert:commit Header(撤销的) SHA

  1. type用于说明 commit 的类别，只允许使用下面7个标识
  2. feat：新功能（feature）
  3. fix：修补bug
  4. docs：文档（documentation）
  5. style： 格式（不影响代码运行的变动）
  6. refactor：重构（即不是新增功能，也不是修改bug的代码变动）
  7. test：增加测试
  8. chore：构建过程或辅助工具的变动
